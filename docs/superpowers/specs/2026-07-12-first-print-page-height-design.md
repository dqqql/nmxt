# 第一页打印高度修复设计

## 问题

浏览器打印统一使用 A4 横向页面，打印容器尺寸为 `1760 × 1245`。第二至第五页同样使用 `1760 × 1245`，因此能够铺满页面；第一页在打印样式中被单独限制为 `1760 × 990`，只占页面高度约 79.5%，导致底部出现约 20.5% 的空白。

## 方案

仅调整打印模式下第一页 `.sheetPageOne` 的固定高度，将其从 `990px` 改为 `1245px`。保持宽度、打印缩放比例和 A4 页面设置不变，由第一页现有的 Grid 弹性行分配新增的纵向空间。

编辑界面的第一页继续使用 `1760 × 990`，第二至第五页保持现状。该修改不会对文字和图形应用非等比缩放。

## 验证

增加样式回归测试，确认打印模式下第一页和 PDF 页面均为 `1245px` 高，并确认屏幕模式下第一页仍为 `990px`。运行完整测试和生产构建，再用 Chromium 打印/PDF 输出检查五页分页数量及第一页底部是否铺满。
