# 引导车卡设计

日期：2026-07-10

## 背景

当前应用是 Vite + React 的静态角色卡工具。主界面展示五页可编辑卡面，右侧工具栏包含“问卷车卡”和“随机生成”。`/wj` 问卷车卡已经通过独立页面、localStorage 草稿、提交结果回填主界面的方式运行。

本次功能新增一个更接近 `D:\Dql\Desktop\tasheet` 的逐步车卡模式。玩家从主界面进入独立的 `/guide` 页面，按步骤手动选择或填写角色信息，最后在竖版预览页确认。确认后回到主界面并展示自动填好的卡面信息。

## 范围

本次实现包含：

- 在主界面右侧工具栏“问卷车卡”下方新增“引导车卡”按钮，点击跳转 `/guide`。
- 新增 `/guide` 独立引导车卡页面。
- 引导步骤顺序为：信息、出身、分配属性、道源、法门、大道、因果值、预览确认。
- 顶部进度条允许玩家在步骤之间跳转，随机生成后也可以回到任一步单独修改。
- 底部提供“上一步 / 下一步”导航；最后一页提供“随机生成”和“确认”。
- 中途退出时保存当前步骤和已填写内容，下次进入同一存档的 `/guide` 继续原进度。
- 不同主界面存档的引导草稿分开保存。
- 确认提交后清空当前存档对应的引导草稿，写入结果并回到主界面。
- 主界面读取结果后回填基础信息、出身、四属性、道源、法门、大道、因果和天赋/天谴。
- 主页原“随机生成”按钮从右侧工具栏移除，移动到引导页最后一页“确认”旁边。
- 随机生成效果保持不变：仍使用现有 `createRandomCardState` 逻辑生成卡面主体，只是结果先展示在引导页预览中。
- 最后预览界面改为竖版排列，尽量将所有引导信息压缩到约 1 到 2 张 A4 纸。

本次不包含：

- 新增后台、账号系统或远程存储。
- 改动现有五页正式卡面的打印导出布局。
- 改动“问卷车卡”的题目、统计规则或 `/wj` 交互。
- 引导页内直接编辑完整五页卡面的所有细节字段。

## 页面入口

主界面右侧工具栏保持现有视觉和交互模式：

- “问卷车卡”按钮保持在原位置，导航到 `/wj`。
- 在“问卷车卡”下方新增“引导车卡”按钮，使用和工具栏一致的竖向按钮尺寸、深色底、白字和图标。
- 移除右侧工具栏中的“随机生成”按钮。

React 入口根据路径渲染：

- `/`：主界面 `App`。
- `/wj`：现有问卷页。
- `/guide`：新增引导车卡页。

## 引导数据结构

新增纯状态模块，例如 `src/guidedCardState.js`，用于测试和复用。核心数据结构：

```js
{
  version: 1,
  step: 0,
  values: {
    name: "",
    race: "",
    belong: "",
    daoHeart: "",
    identity: "",
    origin: null,
    attributes: {
      "仙躯": "",
      "身法": "",
      "神魂": "",
      "灵蕴": ""
    },
    coreAttribute: null,
    source: null,
    method: null,
    dao: null,
    fateValue: 0
  }
}
```

选择类字段保存 option index 或 null，提交结果时转换为主界面可直接使用的 `selections`。`fateValue` 为 `-3` 到 `3`，提交时通过现有 `fateValueToTitle` 转成 `逆命叁`、`逆命贰`、`逆命壹`、`平平无奇`、`天命壹`、`天命贰`、`天命叁`。

引导提交生成的主卡状态与现有随机生成、问卷回填兼容：

```js
{
  version: 1,
  createdAt: "2026-07-10T00:00:00.000Z",
  snapshot: {
    selections,
    texts,
    attributes,
    coreAttribute,
    selectedFateTitle,
    diceEffects,
    drawnTalents,
    thresholdBonuses,
    upgradeChoices,
    maxRealmIndexReached
  }
}
```

## 草稿与存档隔离

使用 localStorage 保存引导草稿和提交结果：

- `nmxt.guidedCard.drafts.v1`：所有存档的引导草稿集合。
- `nmxt.guidedCard.result.v1`：提交后等待主界面消费的结果。

草稿集合按主界面活动存档区分：

- 如果存在 `CARD_ACTIVE_SAVE_SLOT_KEY`，草稿键使用 `slot:<slotId>`。
- 如果没有活动存档，草稿键使用 `slot:default`。

示例：

```json
{
  "slot:default": {
    "version": 1,
    "step": 3,
    "values": {}
  },
  "slot:7d1c": {
    "version": 1,
    "step": 6,
    "values": {}
  }
}
```

进入 `/guide` 时读取当前活动存档对应草稿。每次修改字段或切换步骤时自动保存。点击返回主界面不会清空草稿。点击确认提交后只删除当前草稿键，其他存档草稿保留。

## 步骤交互

`/guide` 页面借鉴 `tasheet` 的结构，但视觉匹配当前项目：

- 顶部为纸面风格步骤条，使用紧凑的方块按钮展示步骤编号和名称。
- 当前步骤深色高亮，已访问步骤使用白底深字，未访问步骤使用浅灰底。
- 页面主体为灰底上的白纸容器，不做营销式首页。
- 底部固定导航包含“上一步 / 当前步骤计数 / 下一步”，最后一页隐藏“下一步”并显示终页操作。

各步骤内容：

1. 信息：填写名称、种族、归属、道心、身份等当前主界面已有 `texts` 字段中与角色身份相关的内容。
2. 出身：从 `originOptions` 中选择一个出身，卡片展示名称、描述和效果。
3. 分配属性：填写仙躯、身法、神魂、灵蕴，支持标记一个核心属性。
4. 道源：从 `sourceOptions` 中选择一个道源，卡片展示描述、阈值、加成、能力和效果摘要。
5. 法门：从 `methodOptions` 中选择一个法门，卡片展示描述、普攻增益和练气感悟摘要。
6. 大道：从 `daoOptions` 中选择一个大道，卡片展示描述和效果摘要。
7. 因果值：选择 -3 到 3 的因果值，显示对应因果标题和会触发的天赋/天谴抽取方向。
8. 预览确认：竖版预览所有信息，提供“随机生成”和“确认”。

选择页使用类似现有资源库的卡片网格，但更适合逐步浏览：左侧或上方为选择列表，右侧或下方为当前选择详情。移动端改为单列。

## 随机生成

最后一页“随机生成”按钮调用现有 `createRandomCardState`：

- 使用 `realmOptions`、`originOptions`、`sourceOptions`、`methodOptions`、`daoOptions`、`fateDraws` 和 `drawByPlan`。
- 将生成结果写回引导页草稿的对应字段。
- 生成后停留在预览页，并显示新的竖版预览。
- 玩家可以通过顶部步骤条回到单独步骤修改随机出的出身、属性、道源、法门、大道或因果值。

随机生成不再出现在主界面工具栏。主界面只有在引导确认或问卷提交后消费对应结果。

## 预览布局

预览页采用竖版 A4 感的滚动预览，不影响正式卡面打印：

- 预览容器宽度接近 A4 竖版比例，屏幕上居中展示。
- 信息以瀑布式方块排列，卡片圆角不超过 8px，边框、浅灰底、黑白纸面为主，少量绿色用于确认状态。
- 第 1 张 A4 优先放：角色信息、四属性、因果、出身、道源、法门、大道。
- 如内容较长，天赋/天谴、道源能力、法门感悟、大道效果可以自然流到第 2 张 A4。
- 文本区域使用固定最小高度和自适应换行，避免按钮或标签被长文本挤爆。
- 预览只展示本次引导负责的关键信息，不复制完整五页正式卡面。

## 主界面接收结果

主界面 `App` 启动时检查 `nmxt.guidedCard.result.v1`：

- 如果存在结果，调用与恢复存档类似的集中函数应用 `snapshot`。
- `texts` 合并到 `defaultTexts`。
- `selections.realm` 固定为练气前期，其他选择来自引导结果。
- `attributes` 来自引导页。
- `coreAttribute` 来自引导页。
- `selectedFateTitle` 来自因果值，`diceEffects` 使用 `getFateState(selectedFateTitle).diceEffects`。
- `drawnTalents` 使用对应因果的抽取方案生成。
- `thresholdBonuses`、`upgradeChoices`、`maxRealmIndexReached` 重置为初始境界状态。
- 清空 `activeSaveSlotId`，避免把引导生成结果误写到当前已打开存档，除非玩家之后主动保存。
- 消费成功后删除结果键并显示 toast：“引导车卡完成！”。

## 错误处理

引导页尽量允许玩家跳步，但确认时做最小校验：

- 出身、道源、法门、大道必须已选择。
- 四项属性必须是可解析的整数；允许 `0`。
- 因果值必须在 `-3` 到 `3`。
- 基础信息可为空。

确认失败时停留在预览页，并在缺失模块上显示错误摘要。点击错误项可跳到对应步骤。

如果 localStorage 不可用，页面仍可在当前会话中填写，但刷新后无法恢复。保存失败不阻断填写，只显示轻量提示。

## Tests

新增 Vitest 覆盖纯状态模块：

- 创建空引导草稿，默认步骤为 0，四属性为空，因果值为 0。
- 更新步骤时限制在有效范围内。
- 根据 active slot id 生成不同草稿键；无 slot 时使用默认键。
- 确认结果能把出身、道源、法门、大道 index 转为主界面 `selections`。
- 因果值 `-3..3` 能正确转为因果标题。
- 随机生成结果能写回引导草稿，并保留可继续修改的结构。
- 提交后只清除当前存档草稿，不删除其他存档草稿。

Build verification:

- `npm test`
- `npm run build`

Manual browser verification:

- 启动 Vite dev server。
- 从主界面点击“引导车卡”进入 `/guide`。
- 填写前几步，返回主界面，再进入 `/guide`，确认恢复到原步骤。
- 切换或创建不同主界面存档后进入 `/guide`，确认草稿互不覆盖。
- 在最后一页点击“随机生成”，确认预览更新，并能通过顶部步骤条回退修改。
- 点击“确认”，确认跳回 `/`，主界面卡面字段已回填，当前引导草稿已清空。
- 确认主界面右侧工具栏不再显示“随机生成”。

## Open Decisions

无开放决策。引导车卡采用独立 `/guide` 页面，终页随机生成，确认后回填主界面。
